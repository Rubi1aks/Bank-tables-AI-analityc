# Контракты API — постранично (для backend, Java)

Документ описывает, **какие запросы делает фронтенд на каждой странице** и в
каком формате ожидает ответ. Источник истины по типам — `src/shared/lib/api-types.ts`,
все сетевые вызовы собраны в `src/shared/lib/api.ts`. Сейчас фронт работает на
моках (MSW, `src/mocks/`); переход на реальный сервер — это смена `BASE_URL` в
`api.ts` и выключение моков. Если backend отдаёт JSON ровно такой структуры —
ничего в компонентах менять не нужно.

---

## Общие соглашения

- **Base URL:** напр. `https://host` (на фронте — `BASE_URL` в `api.ts`, сейчас `''`).
- **Формат:** JSON, UTF-8. Кроме загрузки файла — там `multipart/form-data`.
- **Таймаут:** один синхронный HTTP-запрос ≤ **60 секунд** (иначе фронт прервёт по AbortController).
- **Параллелизм:** тяжёлые задачи (пересчёт сценария) идут через лимитер — не
  более **5 одновременных** запросов; остальные ждут в очереди.
- **Периоды:** в long-формате — `"YYYY-MM"`; в данных «Недели 1» (драйверы) —
  `"YYYY-MM-DD"` (первое число месяца). Внимательно: это разные форматы.
- **Числа:** JSON number с точкой (`31157.6`), без кавычек и разделителей разрядов.
- **Ошибка (4xx/5xx):** JSON `{ "status": <int>, "error": "<текст>" }`. Фронт
  реагирует на HTTP-код; тело используется для показа пользователю.

## Общие модели данных

### FactRow — строка фактов (long-формат, одна широкая таблица, без join)

```json
{ "period": "2026-06", "district": "Центральный", "subject": "Москва",
  "indicator": "Доход банка", "unit": "руб", "value": 31157.6 }
```

| Поле | Тип | Описание |
|---|---|---|
| `period` | string | Отчётный период `"YYYY-MM"` |
| `district` | string | Федеральный округ РФ |
| `subject` | string | Субъект РФ |
| `indicator` | string | Показатель |
| `unit` | string | Мера измерения (`руб`, `чел`, `%`) |
| `value` | number | Значение |

Набор показателей (пример кейса — студенческая столовая): `Студенты`,
`Административный персонал`, `Общее число людей`, `Доля питающихся в столовой`,
`Количество клиентов`, `Средняя стоимость обеда`, `Объём транзакций`, `Тариф`,
`Доход банка`. Связи: `Доход = Объём транзакций × Тариф(5%)`,
`Объём = Клиенты × Средняя стоимость обеда`,
`Клиенты = Общее число людей × Доля питающихся`,
`Общее число людей = Студенты + Адм. персонал`.

### Scenario — сценарий плана

```jsonc
{
  "id": "sc-base",
  "kind": "base",                       // "base"|"optimistic"|"conservative"|"custom"|"ai"
  "title": "Базовый",                   // = params.name, если задан
  "description": "Продление текущих трендов",
  "params": { /* ScenarioParams, см. ниже */ },
  "status": "ready",                    // "queued"|"computing"|"ready"|"failed"
  "series": [ { "period": "2026-07", "value": 132450.0 } ],
  "byRegion": [ { "subject": "Москва", "value": 41230.0 } ],
  "growthRateStd": 0.142,               // метрика качества: СКО темпов роста по субъектам
  "drivers": [
    { "indicator": "Количество клиентов", "contributionPct": 46, "value": 1932, "unit": "чел" }
  ]
}
```

**ScenarioParams:**

| Поле | Тип | Описание |
|---|---|---|
| `name` | string? | Название плана (вводит пользователь) |
| `targetIndicator` | string | Целевой показатель |
| `periodFrom` | number? | Начало отрезка планирования, мес (1..24) |
| `horizonMonths` | number | Конец отрезка планирования, мес (1..24) |
| `seasonality` | boolean | Учёт сезонности (в текущем UI всегда `true`) |
| `method` | enum | `"growth-rate" \| "avg-3m" \| "avg-6m"` |
| `driverMultipliers` | object? | `{ [показатель]: множитель }` (ручной сценарий) |

> Период планирования — **отрезок** `[periodFrom .. horizonMonths]` (напр. 6–12).

## Страница 2. Текущие данные / Обзор (`/`)

Показывает загруженные данные: KPI и динамику дохода + блок «Аналитика по
региону» (выбор субъекта → график драйверов).

### 2.1. `GET /api/facts`

Все факты (long-формат). На больших объёмах желательна серверная пагинация
(см. «Рекомендации»), но базовый контракт — массив `FactRow`.

**Ответ `200`:**
```json
[
  { "period": "2025-07", "district": "Центральный", "subject": "Москва", "indicator": "Доход банка", "unit": "руб", "value": 31157.6 },
  { "period": "2025-08", "district": "Центральный", "subject": "Москва", "indicator": "Доход банка", "unit": "руб", "value": 32140.0 }
]
```

### 2.2. `GET /api/analytics/drivers?subject=<Субъект РФ>`

Драйверы по выбранному региону для графика (оси: X — `reportPeriod`, линии —
`avgArpu` и `marketPenetrationPct`). Без параметра `subject` — вернуть первый
субъект (чтобы график не был пустым).

**Ответ `200` — массив `DriverRow`:**
```json
[
  { "reportPeriod": "2025-01-01", "federalDistrict": "Северо-Западный ФО", "subjectRf": "Мурманская обл.",
    "avgArpu": 13.60, "marketPenetrationPct": 66.98, "avgTransactionCheck": 272.00, "bankIncomeLag1": null },
  { "reportPeriod": "2025-02-01", "federalDistrict": "Северо-Западный ФО", "subjectRf": "Мурманская обл.",
    "avgArpu": 14.80, "marketPenetrationPct": 67.54, "avgTransactionCheck": 275.00, "bankIncomeLag1": 31157.60 }
]
```

| Поле | Тип | Описание |
|---|---|---|
| `reportPeriod` | string | Период `"YYYY-MM-DD"` (1-е число месяца) |
| `federalDistrict` | string | Федеральный округ |
| `subjectRf` | string | Субъект РФ |
| `avgArpu` | number | Средний ARPU |
| `marketPenetrationPct` | number | Проникновение на рынок, % |
| `avgTransactionCheck` | number | Средний чек, руб |
| `bankIncomeLag1` | number \| null | Доход банка со сдвигом на 1 период (у первого = `null`) |

> Список субъектов для выпадающего меню сейчас берётся на фронте из мока.
> Желательно отдавать его отдельным эндпоинтом — см. «Рекомендации».

## Страница 3. Загрузка данных (`/upload`) — только админ

Загрузка Excel/CSV → анимация загрузки (мин. 3 сек) → плашка успеха →
предпросмотр распознанных сущностей и виртуализированная таблица.

### 3.1. `POST /api/analytics/upload` (multipart/form-data)

Поле формы `file` — выбранный Excel/CSV. **Content-Type выставляет браузер**
(boundary), вручную не задавать.

**Ответ `200` — `UploadResultResponse`:**
```json
{ "status": "ok", "message": "Данные успешно загружены и пересчитаны" }
```

| Поле | Тип | Описание |
|---|---|---|
| `status` | string | `"ok"` при успехе |
| `message` | string | Текст для плашки успеха |

> Сейчас обработка синхронная (заглушка отвечает быстро, фронт держит лоадер
> минимум 3 сек). Если реальная обработка файла длится дольше 60 сек — нужен
> асинхронный вариант (загрузка → `uploadId` → прогресс по WebSocket/polling),
> см. «Рекомендации».

### 3.2. `GET /api/entities`

Результат автоопределения сущностей (какая колонка — дата/показатель/…),
показывается на экране разметки после загрузки.

**Ответ `200` — массив `DetectedEntity`:**
```json
[
  { "column": "Отчётный период",      "category": "date",      "confidence": 0.99, "sample": ["2026-04", "2026-05", "2026-06"] },
  { "column": "Федеральный округ РФ", "category": "territory", "confidence": 0.97, "sample": ["Центральный", "Северо-Западный"] },
  { "column": "Субъект РФ",           "category": "territory", "confidence": 0.96, "sample": ["Москва", "Санкт-Петербург"] },
  { "column": "Показатель",           "category": "indicator", "confidence": 0.94, "sample": ["Доход банка", "Студенты"] },
  { "column": "Мера измерения",       "category": "unit",      "confidence": 0.90, "sample": ["руб", "чел", "%"] },
  { "column": "Значение",             "category": "value",     "confidence": 0.98, "sample": ["31157.6", "623152"] }
]
```

`category` ∈ `"date" | "indicator" | "territory" | "value" | "unit"`;
`confidence` — доля 0..1; `sample` — примеры значений колонки.

> Таблица-предпросмотр (>100k строк) сейчас наполняется **синтетикой на фронте**
> для проверки виртуализации; отдельного эндпоинта не требует. При интеграции
> предпросмотр можно брать из `GET /api/facts` (или из пагинированного эндпоинта).

## Страница 4. Граф бизнес-модели (`/graph`)

При входе на канвасе только узлы (связи пользователь строит сам). Кнопка
«Регенерация» заново запрашивает граф из БД.

### 4.1. `GET /api/graph`

**Ответ `200` — `BusinessGraph`:**
```json
{
  "nodes": [
    { "id": "students",  "indicator": "Студенты",          "unit": "чел", "currentValue": 3601, "isDerived": false },
    { "id": "revenue",   "indicator": "Доход банка",        "unit": "руб", "currentValue": 29076.6, "isDerived": false }
  ],
  "edges": [
    { "id": "e-tx-revenue", "source": "txVolume", "target": "revenue", "operator": "*" }
  ]
}
```

**Узел (`nodes[]`):** `id`, `indicator`, `unit`, `currentValue` (number),
`isDerived` (boolean). **Ребро (`edges[]`):** `id`, `source`, `target`,
`operator` ∈ `"+" | "-" | "*" | "/" | "%"`.

> Фронт по умолчанию делает все узлы базовыми (`isDerived:false`) — тип
> «базовая/производная» пользователь переключает сам. Поэтому поле `isDerived`
> из БД на этой странице не критично, но в контракте сохраняется.

## Страница 5. Сценарии (`/scenarios`)

Форма «Новый вариант плана» (название, целевой показатель, период-отрезок,
метод) → генерация сценария; сравнение сценариев; карта регионов; блок
«Расчёт СКО для прогнозируемого дохода (YoY)».

### 5.1. `GET /api/scenarios`

Набор сценариев по умолчанию (3–4 варианта: Базовый/Оптимистичный/
Консервативный/AI).

**Ответ `200` — массив `Scenario`** (структура — см. «Общие модели»).

### 5.2. `POST /api/scenarios/compute`

Пересчёт одного сценария по параметрам формы. **Тяжёлая операция** (лимит 5
параллельных, ≤60 сек).

**Тело запроса (`ScenarioParams`):**
```json
{
  "name": "Базовый Q3",
  "targetIndicator": "Доход банка",
  "periodFrom": 6,
  "horizonMonths": 12,
  "seasonality": true,
  "method": "avg-3m"
}
```

**Ответ `200` — один `Scenario`** со `status: "ready"`, заполненными
`series` (по отрезку `[periodFrom..horizonMonths]`), `byRegion`,
`growthRateStd`, `drivers`. `title` = `name`.

> Допустимо вернуть `status: "queued"`/`"computing"` с пустыми `series`, а
> готовый результат дослать позже — фронт отображает статусы. Базовый вариант —
> синхронный `"ready"`.

### 5.3. Блок «Расчёт СКО для прогнозируемого дохода (YoY)»

Столбчатая диаграмма по регионам: 2024 (синий), 2025 (красный), 2026 (зелёный —
прогноз) + пунктирный доверительный коридор на прогнозе.

> **Сейчас данные берутся из мока на фронте** (`src/mocks/yoyStd.ts`),
> отдельного эндпоинта пока нет. Рекомендуемый контракт для backend:

`GET /api/analytics/yoy-std?confidence=95`

```json
{
  "regions": [
    { "subject": "Москва", "y2024": 484000, "y2025": 542080, "y2026": 583220, "lower": 548227, "upper": 618213 }
  ],
  "sigma": 64210,
  "confidence": 95
}
```

| Поле | Тип | Описание |
|---|---|---|
| `regions[].subject` | string | Субъект РФ |
| `regions[].y2024/y2025` | number | Факт за 2024 / 2025 |
| `regions[].y2026` | number | Прогноз на 2026 |
| `regions[].lower/upper` | number | Границы доверительного коридора прогноза |
| `sigma` | number | Общее СКО суммы |
| `confidence` | number | Уровень доверия, % |

## Страница 7. Экспорт (`/export`)

Выгрузка в Excel формируется **на фронте** (SheetJS): лист «Данные» + лист
«Формула» (граф расчёта). Своих эндпоинтов страница не вызывает, использует
уже полученные данные:

- `GET /api/facts` — строки для листа «Данные» (с фильтром по показателю);
- граф для листа «Формула» — те же данные, что и `GET /api/graph`.

Бэкенду отдельной реализации под экспорт не требуется.

## Рекомендации backend на будущее (не блокируют MVP)

- **Справочники:** `GET /api/subjects`, `GET /api/indicators`, `GET /api/periods`
  — чтобы выпадающие списки не строились из всего массива фактов/моков.
- **Серверная пагинация фактов:**
  `GET /api/facts?subject=&indicator=&period=&offset=&limit=` →
  `{ "rows": FactRow[], "total": number }` (для таблиц >1 млн строк).
- **Асинхронная загрузка** (если обработка файла > 60 сек):
  `POST /api/analytics/upload` → `{ "uploadId": "..." }`, затем прогресс по
  WebSocket `/ws/upload/{uploadId}` или `GET /api/analytics/upload/{uploadId}/status`.
- **Эндпоинт YoY-СКО** (`/api/analytics/yoy-std`) — перевести блок СКО с мока
  на реальные данные.
